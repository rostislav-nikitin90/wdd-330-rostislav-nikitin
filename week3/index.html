Chapter 5: Objects

Notes:

An object in JavaScript is a set of properties, and each property consists of a name and a value associated with that name. The value of a property can be any JavaScript value. If it's a function, it's known as a method of an object. An object literal is an object that is created directly in the language by wrapping all its properties and methods in curly braces {}. The creation of an object literal begins with the definition and initialization of the variable to which the object literal is assigned. The properties and methods of an object can be accessed by using dot notation or square brackets.

New properties and methods can be added to objects at any time in a program by simply assigning a value to the new property. An object's properties and methods can be changed or removed because objects are mutable. Objects placed inside another object are called nested objects. The values in nested objects can be accessed by referencing each property name in order using either dot or bracket notation.

Objects can be used as parameters to functions, which allows arguments to be entered in any order, or omitted.

When an object method needs to access information stored in an object to do its job, it uses the “this” keyword to access the object. The value of this is the object “before dot”, the one used to call the method. Example:

let user = {

name: "John",

age: 30,

sayHi() {

// "this" is the "current object"

alert(this.name);

}

};

user.sayHi(); // John

Here during the execution of user.sayHi(), the value of this will be user.

JavaScript Object Notation, or JSON, is a portable data format that uses JavaScript object literals to exchange information. JSON data is written as name/value pairs, just like JavaScript object properties. There are, however, a couple of key differences:

1. Property names must be double-quoted

2. Permitted values are double-quoted strings, numbers, true, false, null, arrays and objects

3.Functions are not permitted values

Example of JSON string representation:

const batman = '{"name":"Batman","real name":"Bruce Wayne","hero":true,"villain":false}';

The parse() method takes a string of data in JSON format and returns a JavaScript object:

JSON.parse(batman);

<< { name: 'Batman',

'real name': 'Bruce Wayne',

height: 74,

weight: 210,

hero: true,

villain: false,

allies: [ 'Robin', 'Batgirl', 'Superman' ] }

The stringify() method does the opposite, taking a JavaScript object and returning a string of JSON data, as can be seen in the example:

JSON.stringify(wonderWoman);

<< '{"name":"Wonder Woman","real name":"Diana Prince","hero":true,"villain":false}'

The Math object is a built-in object that has several properties representing mathematical constants, as well as methods that carry out a number of common mathematical operations. The Math object can be used to perform mathematical calculations.

The Date object can be used to create date objects. Date objects contain information about dates and times. Each object represents a single moment in time.

The properties of date objects are unable to be viewed or changed directly. Instead, they have a number of methods known as getter methods, which return information about the date object, such as the month and year. Once the date object has been created, it will have access to all getter methods.

Most of the getter methods have equivalent setter methods. These are methods that can be used to change the value of the date held in a Date object. Each of the methods takes an argument representing the value to which the date is updated. The methods return the timestamp of the updated date object.

The Regex object can be used to create regular expressions.

Chapter 6: The Document Object Model

Notes:

The Document Object Model, or DOM for short, represents an HTML document as a network of connected nodes that form a tree-like structure. The DOM treats everything on a web page as a node, where each node is an object of the document. HTML tags, the text inside these tags, even the attributes of a tag are all nodes. The DOM also stores any whitespace that is in the HTML document as text nodes. The HTML tag is the root node, and every other part of the document is a child node of this. When a web page is loaded, the browser creates a Document Object Model of the page. As an object-oriented representation of the web page, it can be modified with a scripting language such as JavaScript to access and modify different parts of a web page using a special built-in object called document.

The DOM provides several methods that allow to access any element on a page. These methods will return a node object or a node list, which is an array-like object. These objects can then be assigned to a variable and be inspected or modified. An example of accessing the body element of a web page and assigning it to the variable body:

const body = document.body;

Now it shows a reference to the body element, allowing to check its type:

typeof body;

<< "object";

This is a special Node object with a number of properties and methods that can be used to find information about, or modify, the body element.

For example, the nodeType property can be used to find out what type of node it is:

body.nodeType;

<< 1

All nodes have a numerical code to signify what type they are. These are summmarized in the table below:

Code Type
1 element
2 attribute
3 text
8 comment
9 body

There are other types not covered in the table, but these aren't used in HTML documents. As can be seen from the table, a code of 1 confirms that body is an element node.

The nodeName property can also be used to find the name of the element:

body.nodeName;

<< "BODY"

The document.getElementById(), document.getElementsByClassName(), document.getElementsByTagName() and document.querySelector() can be used to access elements on a page.

The document.getElementById() method returns a reference to the element with a unique specified id attribute that is given as an argument. An example of referencing to the ‹h1› heading element with the id of 'title' in an html page by writing this in the console:

const h1 = document.getElementById('title');

The getElementsByTagName() method returns a live node list of all the elements with the tag name that is provided as an argument. An example of getting all the list items (HTML tag of ‹li›) in the document:

const listItems = document.getElementsByTagName('li');

The getElementsByClassName() method returns a live node list of all elements that have the class name that is supplied as an argument. An example of returning a collection of all elements with the class of 'hero':

const heroes = document.getElementsByClassName('hero');

The document.querySelector() method allows to use CSS notation to find the first element in the document that matches a CSS selector provided as an argument.

The document.querySelectorAll() method also uses CSS notation but returns a node list of all the elements in the document that match the CSS query selector. If no elements match, it will return an empty node list.

The parentNode(), previousSibling(), nextSibling(), childNodes() and children() methods can be used to navigate around the DOM tree.

All HTML elements have a large number of possible attributes such as 'class', 'id', src, and 'href'. The DOM has numerous getter and setter methods that can be used to view, add, remove or modify the value of any of these attributes.

The getAttribute() method returns the value of the attribute provided as an argument. Example:

wonderWoman.getAttribute('class');

<< "hero"

The setAttribute() method can change the value of an element's attributes. It takes two arguments: the attribute that the programmer wishes to change, and the new value of that attribute. An example of changing the the class of the element in the wonderWoman variable to 'villain':

wonderWoman.setAttribute('class', 'villain');

<< undefined

wonderWoman.getAttribute('class');

<< "villain"

The createElement() method creates the HTML element specified by tagName. It takes a tag name as a parameter and returns that element. An example of creating a new list item as a DOM fragment in memory:

const flash = document.createElement('li');

The createTextNode() method creates a text node with the specified text. It takes a parameter, which is a string containing the text that goes in the node. Example:

const flashText = document.createTextNode('Flash');

To link the created list element node to the created text node, it is necessary to append the text node to the list element node using the appendChild() method that will append a node (given as an argument) as the last child of a particular parent node. In this case a newly created text node will become a child node of the list element node. This means that it's the flash object that calls the method, with flashText as its argument:

flash.appendChild(flashText);

The result gives a ‹li› element that contains the text 'Flash'.

Elements can be replaced using the replaceChild() method, and removed using the removeChild() method.

The innerHTML property returns all the child elements of an element as a string of HTML. If an element contains lots of other elements, all the raw HTML is returned. Example:

heroes.innerHTML

<<"

‹li class=\"hero\"›Superman‹/li›

‹li class=\"vigilante hero\" id=\"bats\"›Batman‹/li›

"

Chapter 7: Events

Notes:

Events occur when a user interacts with a web page, such as clicking a link, pressing a key, or moving the mouse.

An event listener is a procedure or function in a computer program that waits for an event to occur. It is like setting a notification to alert the user when something happens. Instead of the program having to constantly check to see if an event has occurred, the event listener will let it know when the event happens, and the program can then respond appropriately. This allows the program to continue with other tasks while it waits for the event to happen.

An event listener is attached to an element, then invokes a callback function when the event occurs. In the example below, an event listener has been added to the document's body element:

document.body.addEventListener("click", doSomething);

It will call the function doSomething() when any part of the page is clicked on. Until that happens, the program will continue to run the rest of the code. The addEventListener() method is called on a node object, the node to which the event listener is being applied. In the example above, the code will attach an event listener to the document’s body.

There are many types of event, including mouse events, keyboard events, and touch events.

The removeEventListener() method removes an event listener that has been attached with the addEventListener() method. Example:

Code in HTML:

‹p id='once'›A One Time Thing...‹/p›

JavaScript:

const oncePara = document.getElementById('once');

oncePara.addEventListener('click', remove);

function remove(event) {

console.log('Enjoy this while it lasts!');

oncePara.style.backgroundColor = 'pink';

oncePara.removeEventListener('click', remove);

}

This adds a click event listener to a paragraph element, but then removes it in the callback function named remove. This means it will only be called once (if the user tries to click on it again, nothing will happen).

The default behavior of elements can be prevented using the preventDefault() function.

Event propagation is the order that the events fire on each element. There are two forms of event propagation: bubbling and capturing. Bubbling is when the event fires on the element clicked on first, then bubbles up the document tree, firing an event on each parent element until it reaches the root node. Capturing starts by firing an event on the root element, then propagates downwards, firing an event on each child element until it reaches the target element that was clicked on.

Event delegation is used to attach an event listener to a parent element in order to capture events that are triggered by its child elements.

    // View Object

    const view = {

    }

    target.innerHTML = content;

    }

    };

    This uses the document.querySelector() method to access the desired elements and assign them to a variable. So, for example, the div with an id of question can be accessed in the Javascript code using view.question.

This function has three parameters: the first is the element that displays the content, the second is for the content it's to be updated with, and the last is an object of any HTML attributes that can be added to the element.

    The function loops through any attributes provided as the third argument, and uses the setAttribute() method to update them to the values provided. It then uses the innerHTML property to update the HTML with the content provided.

    ask() {
